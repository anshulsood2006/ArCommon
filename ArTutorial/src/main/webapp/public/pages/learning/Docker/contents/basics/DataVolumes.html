Data Volumes in Docker

Features

  • Data volumes are special directories in docker containers which are initalized when containers are created.
  • A data volume is not deleted when containers are removed.
  • A data volume is never garbage collected.
  • A data volume can be shared across containers.
  • A data volume is created using command "docker run -it -v /data -name {containerAlias} {imageName}".
  • To remove data volume when container is removed, use command: docker container -rm {containerName} -v.
  • To map a host machine folder to data volume use command: docker run -it -v {hostFolderPath:containerFolderPath} -name {containerAlias} {imageName}
  • To get the data volume mounted on one container inside another container, launch the other container using command: docker run -it --volumes-from {containerAliasWhoseVolumeNeedsToBeShared} -name {containerAlias} {imageName}
