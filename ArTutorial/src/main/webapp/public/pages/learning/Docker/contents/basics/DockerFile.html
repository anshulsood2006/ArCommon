Docker File

Features

  • Docker file is used to create user defined docker images.
  • It is a simple text file with name Dockerfile and without any extension.
  • It consists ofa series of layers built on top of each other, starting with a base image.
  • Base image can be an existing docker image or scratch.
  • Once a docker file is created, image can be created using command : docker build -t {imageName:version} {path of Dockerfile}

Structure of docker file

  • FROM Used to specify the base image. Should be scratch if no base image needs to be used.
  • MAINTAINER Used to specify the creator of image.
  • USER Used to specify the user to be used while running the image and for any RUN, CMD and ENTRYPOINT instructions that follow it.
  • WORKDIR Used to create and move to working directory during building image..
  • RUN Used to specify the commands to be run during building image.
  • CMD Used to specify the commands to be run during running container.

Examples

Sample Dockerfile

	FROM ubuntu:latest
	MAINTAINER Anshul Sood <anshulsood2006@gmail.com>
	RUN apt-get update
	CMD ["ls"]
