Docker for Mysql

Steps to create MySql container using docker client

  • To start MySql container with root password mypassword at port 3306 on local machine with name test-mysql use command:
    docker run --detach --name=test-mysql --env="MYSQL_ROOT_PASSWORD=mypassword" --publish 3306:3306 mysql
  • To pass configuration options such as max-connections as flags use command:
    docker run --detach --name=test-mysql --env="MYSQL_ROOT_PASSWORD=mypassword" --publish 3306:3306 mysql --max-connections=200
  • To mount a configuration file from host machine's directory say E:\ext\mysqlconf to a directory inside container say /etc/mysql/conf.d use command:
    docker run --detach --name=test-mysql --env="MYSQL_ROOT_PASSWORD=mypassword" --publish 3306:3306 --volume=E:\ext\mysqlconf:/etc/mysql/conf.d mysql
    This can be verified by inspecting container and this will show 
    "HostConfig": {
           "Binds": [
               "/host_mnt/e/ext/mysqlconf:/etc/mysql/conf.d"
           ]
  • To mount data directory from host machine's directoy say E:\ext\mysqldata to the default mysql data directory inside a docker container viz. /var/lib/mysql use command:
    docker run --detach --name=test-mysql --env="MYSQL_ROOT_PASSWORD=mypassword" --publish 3306:3306 --volume=E:\ext\mysqlconf:/etc/mysql/conf.d --volume=E:\ext\mysqldata:/var/lib/mysql mysql
    This can be verified by inspecting container and this will show 
    "HostConfig": {
           "Binds": [
               "/host_mnt/e/ext/mysqldata:/var/lib/mysql",
               "/host_mnt/e/ext/mysqlconf:/etc/mysql/conf.d"
           ]
  • To stop MySql container with name test-mysql use command:
    docker stop test-mysql
  • To remove MySql container with name test-mysql use command:
    docker rm test-mysql
