Dependency Injection (Wiring)

  • It is a way to achieve inversion of control (IoC) in an application by moving objects binding from compile time to runtime.
  • It is achieved by
    • Designing service classes using interfaces or abstract classes.
    • Designing consumer classes using interfaces or abstract classes.
    • Creating injector classes to initialize the service and consumer classes.
  • It makes the application easily extendable.
  • It is also called wiring

Configuring a spring based application

  • Add all the required spring libraries to the project
  • Go to the web.xml file inside web-app/WEB-INF and add the entry for spring's dispatcher servlet as 
    <servlet>
	<servlet-name>SpringServlet</servlet-name>
	<servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
	<load-on-startup>0</load-on-startup>
</servlet>
  • Beans configuration
    • Application Level Beans
      • To define the beans which are available for the whole application context, we need to add ContextLoadListener as below 
        <listener>
	<listener-class>org.springframework.web.context.ContextLoaderListener</listener-class>
</listener>
      • Then we need to configure the parameter contextConfigLocation as context-param 
        <context-param>
	<param-name>contextConfigLocation</param-name>
	<param-value>/WEB-INF/springs/custom.xml</param-value>
</context-param>
    • Servlet Specific Beans
      • The servlet specific beans are by default searched inside file with name {DispatcherServletName}-servlet.xml inside folder WEB-INF
      • This file is used to define the beans to be used in ServletConfig context i.e. for that particular servlet only.
      • The name of the file as well the location can be changed using the same init-param like below for the servlet 
        <servlet>
	<servlet-name>SpringServlet</servlet-name>
	<servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
	<init-param>
		<param-name>contextConfigLocation</param-name>
		<param-value>/WEB-INF/springs/custom.xml</param-value>
	</init-param>
	<load-on-startup>0</load-on-startup>
</servlet>
  • If we have multiple bean files inside resources folder of the project, they can be added to context-param in web.xml file as below 
    <context-param>
	<param-name>contextConfigLocation</param-name>
	<param-value>
		classpath:file1.xml
		classpath:file2.xml
		classpath:file3.xml
	</param-value>
</context-param>

What is the difference between DTO and DAO?


    
	
  • DTO stands for Data Transfer Objects and are used to transfer data between classes and modules of an application. They do not contain any business logic and contain only variables, getters, setters and constructors.
    • 
	
  • DAO stands for Data Access Objects and are used to encapsulated the logic for CRUD operations on data storage.
