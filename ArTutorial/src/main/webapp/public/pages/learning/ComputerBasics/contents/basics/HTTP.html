HTTP

HTTP Request

  • It consists of below four parts
    • Request Line e.g. GET /news.asp HTTP/1.1 It consists of three parts
      • HTTP method type
      • URI for the requested resource
      • Protocol version
    • Message Headers These are colon separated name-value pairs in clear-text string format and provide information about the request objects being sent in the message body
    • Empty Line
    • Message Body It is used to carry the data associated with the request, including some request-related parameters.
    • Example 
      POST /news.asp HTTP/1.1
Host: demo.com:80
Content-Length: 15

[a=1,(b=2,c=3)]

HTTP Response

  • It consists of below four parts
    • Status It gives the http status code for the response message
    • Message Headers These are colon separated name-value pairs in clear-text string format and provide information about the response objects being sent in the message body
    • Empty Line
    • Message Body It is used to carry the data associated with the response.

HTTP Methods

  • HTTP request supports the below methods in the request line
    • GET is used to retrieve information from the server.
    • HEADis used to transfer the status line and header section only.
    • POST is used to send data to the server.
    • PUT is used to update the target resource with the uploaded content. It is idempotent.
    • DELETE is used to delete the target resource.
    • CONNECT is used to establish a tunnel to the server.
    • OPTIONS is used to describes the communication options for the target resource.
    • TRACE is used to performs a message loop-back test along the path to the target resource
  • All the above methods are case sensitive.

HTTP Headers

HeaderDescription
General-headers: These are available for both request and response
cache-control It is used to specify parameters for the caching of documents from the cache. Values can be
  • Request
    • no-cache
    • no-store
    • max-age = seconds
    • max-stale [ = seconds ]
    • min-fresh = seconds
    • no-transform
    • only-if-cached
  • Response
    • public
    • private
    • no-cache
    • no-store
    • no-transform
    • must-revalidate
    • proxy-revalidate
    • max-age = seconds
    • s-maxage = seconds
pragma It is for backwards compatibility with HTTP/1.0 clients where the Cache-Control HTTP/1.1 header is not yet present. Example values are
  • no-cache
connection It is used to specify options that are desired for that particular connection. Example values are
  • connection
  • close
  • keep-alive
date This is in GMT. Example values are
  • Sun, 06 Nov 1994 08:49:37 GMT
  • Sunday, 06-Nov-94 08:49:37 GMT
  • Sun Nov 6 08:49:37 1994
Request specific: These are applicable for requests only
accept It is used to specify the media types which are acceptable. When a q value is given, the media type without q value are highest preferable followed by media type with higher q value (range is between 0 and 1). Example values are
  • text/plain; q=0.5, text/html, text/x-dvi; q=0.8, text/x-c
  • *
accept-charset It is used to specify the character set which are acceptable. When a q value is given, the character set without q value are highest preferable followed by character set with higher q value (range is between 0 and 1). Example values are
  • iso-8859-5, unicode-1-1; q=0.8
  • *
accept-encoding It is used to specify the content-codings which are acceptable. When a q value is given, the content-codings without q value are highest preferable followed by content-codings with higher q value (range is between 0 and 1). Example values are
  • compress;q=0.5, gzip;q=1.0
  • *
accept-language It is used to specify the natural language which are acceptable. When a q value is given, the natural language without q value are highest preferable followed by natural language with higher q value (range is between 0 and 1). Example values are
  • da, en-gb;q=0.8, en;q=0.7
  • *
authorization It consists of authorization parameter is the string of username:password encoded in base 64. Example values are
  • Z3Vlc3Q6Z3Vlc3QxMjM
cookie It contains a name/value pair of information stored for that URL. Example values are
  • name1=value1;name2=value2;name3=value3
expect It is used to indicate that a particular set of server behaviors is required by the client else it will return 417 response code. Example values are
  • expectation-extension
  • 100-continue
from It is used to indicate the e-mail address for the human user who controls the requesting user agent. Example values are
  • anshulsood2006@gmail.com
host It is used to specify the Internet host and the port number of the resource being requested. Example values are
  • 10.12.13.25:7654
  • www.myfriendwa.com
if-match It is used to perform a precondition match. This header requests the server to perform the requested method only if the given value in this tag matches the given entity tags represented by ETag else it will return 412 response code. Example values are
  • "xyzzy"
  • *
if-none-match It is used to perform a precondition match. This header requests the server to perform the requested method only if the none of the given value in this tag matches the given entity tags represented by ETag. Example values are
  • "xyzzy"
  • *
if-modified-since It is used to return the requested resource only if there is any modification since the specified date else it will return 304 response code. Example values are
  • Sat, 29 Oct 1994 19:43:31 GMT
if-unmodified-since It is used to return the requested resource only if there is no modification since the specified date. Example values are
  • Sat, 29 Oct 1994 19:43:31 GMT
range It is used to specify the range of content requested from the document.Example values are
  • bytes=0-499 (First 500 bytes)
  • bytes=-500 (Last 500 bytes)
  • bytes=0-0,-1 (First and last byte only)
referer It is used to specify the address (URI) of the resource from which the URL has been requested. Example values are
  • http://www.myfriendwa.com/student.htm
user-agent It is used to specify the information about the user agent originating the request. Example values are
  • Mozilla/4.0 (compatible; MSIE5.01; Windows NT)
Response specific: These are applicable for response only
accept-ranges It is used to indicate the acceptance of range requests for a resource of a server. Example values are
  • bytes
  • none
age It gives the amount of time since the response was generated at the server. Example values are
  • 1030
etag It gives the current value of entity tag for the requested resource. Example values are
  • W/"xyzzy"
location It is used to redirect the recipient to a location other than the Request-URI for completion. Example values are
  • http://www.myfriendwa.com/index.htm
retry-after It is used to indicate how long the service is expected to be unavailable to the requesting client. Example values are
  • Fri, 31 Dec 1999 23:59:59 GMT
  • 120
server It is used to give information about the software used by the origin server to handle the request. Example values are
  • Apache/2.2.14 (Win32)
set-cookie It contains a name/value pair of information to retain for this URL. Example values are
  • name1=value1,name2=value2;comment=comment;domain=www.myfriendwa.com;path=/pages;secure;Expires=Wed, 09 Jun 2021 10:18:14 GMT
allow It is used to list the set of methods supported by the resource identified by the Request-URI. Example values are
  • GET, HEAD, PUT
content-encoding It is used as a modifier to the media-type. Example values are
  • gzip
content-language It describes the natural language(s) of the intended audience for the enclosed entity. Example values are
  • mi, en
content-length It is used to indicate the size of the entity-body, in decimal number of OCTETs, sent to the recipient. Example values are
  • 3495
content-location It is used to supply the resource location for the entity enclosed in the message when that entity is accessible from a location separate from the requested resource's URI.. Example values are
  • http://www.myfriendwa.com/index.htm
content-MD5 It is used to supply an MD5 digest of the entity for checking the integrity of the message upon receipt. Example values are
  • 8c2d46911f3f5a326455f0ed7a8ed3b3
content-range It is sent with a partial entity-body to specify where in the full entity-body the partial body should be applied. Example values are
  • bytes 0-499/1234
content-type It is used to indicate the media type of the entity-body sent to the recipient. Example values are
  • text/html; charset=ISO-8859-4
expires It gives the date/time after which the response is considered stale. Example values are
  • Thu, 01 Dec 1994 16:00:00 GMT
last-modified It is used to indicate the date and time at which the origin server believes the variant was last modified. Example values are
  • Tue, 15 Nov 1994 12:45:26 GMT

HTTP Status Code

CodeDescription
304If there is no modification in the requested resource since the date specified in if-modified-since request header.
412If the pre-condition specified inside if-match request header fails.
415If the content-coding of an entity in a request message is not acceptable to the origin server
417If the server behavior is not as per the expect request header.
