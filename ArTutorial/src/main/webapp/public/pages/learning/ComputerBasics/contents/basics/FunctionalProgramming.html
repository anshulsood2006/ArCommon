Functional Programming

Features

  • Everything is a mathematical function.
  • The programs are executed by evaluating expressions inside functions.
  • There is a complete separation between the data of a program, and the behaviors of a program
  • All objects created in functional programming are immutable.
  • Shared state is avoided.
  • Adherence to pure functions.

Pure functions

A pure function is a function where:
  • The return value only depends on the input. An input value always return the same value.
  • There are no side effects which means there are no network or database calls which could alter the output.
  • The input data is immutable.

Higher Order Function:

A higher order function is a function which:
  • takes a functions as an input.
  • returns a function as an output.

Tuple

  • A tuple is a compound data type having a fixed number of terms.
  • Each term in a tuple is known as an element.
  • The number of elements is the size of the tuple.
  • It is fixed in size and is immutable.
  • It can contain elements of different types.
  • It is denoted by round parenthesis around the elements e.g. (1, 2, 3, "Anshul").

Record

A record is a data structure for storing a fixed number of elements.

Lazy evaluation/Curring

It is an evaluation strategy which holds the evaluation of an expression until its value is needed

Examples

Impure functions

	function number(num){
	  	num * Math.random()
	}
	The above function gives different result everytime.
	
	function hello(greeting){
	  	console.log(greeting)
	}
	The above function creates a side effect which is console logging.
	 
	var totalPeople = 10
	function totalVotes(votes){
	 	return votes * totalPeople
	}
	The above function depends on a variable outside the function.
