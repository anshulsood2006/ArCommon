Features

Transaction

  • It is a logical unit of work that either completely succeeds or produces no result at all.
  • Distributed transaction:
    • It is a transaction that accesses and updates data on two or more networked resources, and therefore must be coordinated among those resources.
    • It is also called an XA transaction.

Distributed Transaction Processing Model

  • It consists of below components:
    • Application: The end user/component that sends a request to or receive data from database.
    • Application Server: It is a component in distributed network which handles the business logic for the application.
    • Resource adapter: The component that acts as the communication channel between application and the resource manager e.g. jdbc driver.
    • Resource Manager: The component that handles the actual data.
    • Transaction Manager: The component that handles the coordination between different resource managers in case of distributed network.
