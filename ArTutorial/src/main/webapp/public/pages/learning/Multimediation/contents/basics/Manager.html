Multimediation Manager

Configuration

  • When a manager application starts it reads the web.xml file inside WEB-INF.
  • There are 3 listeners defined for the manager
    • org.springframework.web.context.ContextLoaderListener This listener initializes all the beans defined in the xml files which are referred in the context-param for parameter contextConfigLocation
      • applicationContext.xml Beans are
        • com.ericsson.cm.manager.beanfactory.BeanManager
          • This class implements ApplicationContextAware interface and is used to get ApplicationContext to get defined beans anywhere in the manager code
        • com.ericsson.cm.manager.filesystem.FileSystemManager
        • com.ericsson.cm.manager.filesystem.RemoteFileExecutor
        • com.ericsson.cm.manager.usermanagement.model.UserManagerImpl
        • com.ericsson.cm.manager.configurationmanager.ConfigManagerImpl
          • There are some beans which are hibernate Dao objects e.g. HibernateAlarmViaEmailConfigDAO.
          • The DAO classes are using hibernateTemplate bean instead of session factory which is not recommended and are annotated with @Repository annotation having bean name as an argument. This name is directly being referred in the beans configuration files.
          • The DTO classes are classes having table structures defined as class variables.
          • The mapping between the DTO classes and corresponding database tables is done in corresponding hibernate's .hbm.xml files inside resources folder.
          • The path to all hibernate mapping files are given as property mappingLocations of sessionFactory bean.
      • dataManagementContext.xml
      • applicationContextTracer.xml
      • FileMediationDataMgmtContext.xml
      • Beans like datasource, transaction manager, sessionFactory, hibernateTemplate beans all are defined in these xml files only
      • The hibernate mapping files corresponding to Dao classes
    • org.springframework.web.context.request.RequestContextListener
    • com.ericsson.cm.manager.sessionmanager.core.model.SessionManagerExecutorServiceListener
  • The instances of any bean defined in any of the bean configuration files can be created by calling BeanManager.getBean({"beanName"}) from anywhere in the code.
    • Bean Manager class is a class extending ApplicationContextAware interface and is used to get the beans from the ApplicationContext anywhere in the code.
    • This class overrides setApplicationContext() method of the interface.
