Scala

Features

  • Its stands for Scalable Language.
  • It is a hybrid language and has the features of both the object oriented and functional programming language.
    • Object oriented since every value is an object.
    • Functional since every function is a value.
  • It was created by Martin Odersky.
  • It boosts development productivity, applications scalability and overall reliability.
  • It provides a lightweight syntax for defining anonymous functions.
  • It supports higher-order functions which are the functions that accept function as arguments and return function as a result.
  • It allows functions to be nested.
  • It supports currying which is lazy evaluation.
  • It is statically typed that is the type of variable is known at the compile time. It does not expect you to provide redundant type information.
  • It is compiled into Java Byte Code which is executed by the Java Virtual Machine (JVM).
  • It allows to write codes in an immutable manner, which makes it easy to apply concurrency and parallelism.
  • It gives an illusion of an interpreted language. But actually it is a compiled language, wherein everything you type gets compiled to the byte code and it runs within the JVM.
  • It is case sensitive.
  • Scala program processing starts from the main() method which is a mandatory part of every Scala Program. The signature of main method is 
    	def main(args: Array[String])
  • It is a line-oriented language where statements may be terminated by semicolons (;) or newlines.

Scala Identifiers

The names used for classes, variables and methods are called identifiers. The supported are

  • Alphanumeric Identifiers e.g. name, _price
  • Operator Identifiers e.g. + ++ ::: <?> :>
  • Mixed Identifiers e.g. unary_+, myvar_=
  • Literal Identifiers e.g. `anshul`
Note: $ is reserved keyword and should not be used.

Scala Closure

It is a function, whose return value depends on the value of one or more variables declared outside this function.

Scala Trait

It encapsulates method and field definitions, which can then be reused by mixing them into classes.

Examples

Sample scala program to print Hello World

object HelloWorld {
   def main(args: Array[String]) {
      println("Hello, world!")
   }
}
  • Save the file with the above content as HelloWorld.scala
  • Compile using scalac HelloWorld.scala
  • Run using command scala HelloWorld

Frequently Asked Questions

How can we import more than one classes from the same package in scala?

This can be done using below import statement: 

import scala.collection.immutable.{TreeMap, TreeSet}

What are the biggest syntactical differences between scala and Java?

  • In scala the line end character i.e. ';' is optional.
  • There are no primitive types in scala like in Java. All the data types are objects and methods can be called on them.
