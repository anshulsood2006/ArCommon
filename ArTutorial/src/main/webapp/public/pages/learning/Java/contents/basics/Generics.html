Generics

Features

  • It was introduced in Java 1.5
  • It provided the compile time type safety to collections and helps to eliminate unnecessary type casting. e.g.
    • List<Integer> myIntList = new LinkedList<Integer>(); will create a list of Integer objects.
    • List<T> myIntList = new LinkedList<T>(); will create a list of any kind of objects.
  • The abbreviations used most commonly are:
    • E for element
    • T for type
    • K for key
    • V for value
    • N for number
  • PECS
    • PECS stands for "Producer Extends, Consumer Super" which means if you want a list to produce T values, use <? extends T>, whereas if you want a list to consume T values, use <? super T>
    • <? extends T> means that the reference variable can hold any value from a family of type T. So we can retrieve back an item of type T from this list but when we try to add an item of type T to the list, we get a compilation error since the compiler won't be able to identify what type of list this reference variable is pointing to. We can add null though.
    • <? super T> means that we can not retrieve back an item of type T from this list because when we try to do that compiler won't be able to determine what type of list the reference variable is pointing to. But we can get value of type Object. When we add a value of type T, this can be done without any compilation issue.
  • Get and Put Principle.
    • Use an extends wildcard when you only get values out of a structure.
    • Use a super wildcard when you only put values into a structure.
    • And don't use a wildcard when you both get and put.
