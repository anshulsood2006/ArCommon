Decorator Design Pattern

Features

It is a type of structural design pattern

  • We can change the functionality of objects at runtime
  • This is achieved in below manner
  • Define component interface.
  • Define component interface implementation class.
  • Define decorator class having HAS-A relationship with the component interface.
  • Define concrete decorator classes.
  • Decorator pattern is used a lot in Java IO classes, such as FileReader, BufferedReader etc.

Examples

Component Interface: 

public interface Car {
	public void assemble();
}
Component Interface Implementation: 
public class CarComponentImplementation implements Car{

	@Override
	public void assemble() {
		System.out.println("Basic Car");		
	}

}
Decorator: 
public class CarDecorator implements Car{
	protected Car carComponent;
	
	public CarDecorator(Car carComponent){
		this.carComponent = carComponent;
	}

	@Override
	public void assemble() {
		this.carComponent.assemble();		
	}
}
Concrete Decorators: 
public class LuxoryCar extends CarDecorator{

	public LuxoryCar(Car carComponent) {
		super(carComponent);
		System.out.println("Luxory Car");
	}

}

public class SportsCar extends CarDecorator{

	public SportsCar(Car carComponent) {
		super(carComponent);
		System.out.println("Sports Car");
	}

}
Test the decorator pattern: 
public class DecoratorPatternTest {
	public static void main(String[] args) {
		
		Car car = new SportsCar(new LuxoryCar(new CarComponentImplementation()));
		car.assemble();
	}
}
