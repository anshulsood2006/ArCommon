Method Overloading

Features

  • Methods overloading is also called static polymorphism or compile time binding.
  • The two methods are said to be overloaded if the method name is same but arguments are different e.g.
    1. No of arguments e.g.
      • public int add(int, int)
      • public int add(int, int, int)
    2. Type of arguments e.g.
      • public int add(int, int)
      • public int add(int, float)
    3. Order of arguments e.g.
      • public int add(int, float)
      • public int add(float, int)
  • The two methods are NOT said to be overloaded if only the return type is different e.g.
    • public float add(int, int)
    • public int add(int, int)
    This will give compilation error: "Duplicate method in type"
  • Methods can not be overloaded by just adding keyword static. e.g. below code will give compilation error: "Duplicate method in type"
    	public String method(){
		System.out.println("method called");
		return "method";
	}
	
	public static String method(){
		System.out.println("method called");
		return "method";
	}
  • Type Promotion: When a data type of smaller size is promoted to the data type of bigger size than this is called type promotion, for example: byte data type can be promoted to short, a short data type can be promoted to int, long, double etc.
