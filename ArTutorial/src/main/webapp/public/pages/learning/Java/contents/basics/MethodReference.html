Method Reference

Features

  • It is used to refer to static or non static methods or a constructor of a class.
  • It is a shorthand syntax for a lambda expression that executes just ONE method. Syntax is: Object :: methodName.
  • It is used to pass just a reference of a method as an argument to another method instead of passing the complete object.
  • It can only be used to replace a single-method lambda expression where all your lambda expression does is just calling another method with the parameters passed to the lambda.
  • It is of three types:
    1. Static Method Reference. Syntax is: Class::staticMethod
    2. Instance Method Reference for an object of a particular type. Syntax is: ObjectType::instanceMethod
    3. Instance Method Reference for an existing object. Syntax is: object::instanceMethod
    4. Constructor Reference. Syntax is: ClassName::new

Examples

Static Method Reference

The method in the interface and the static method in the class should have the equivalent arguments. 

public abstract interface MyInterface{
	public void draw(String color, int shade);
}

public class StaticMethodReference {

	public static void drawPainting(String color, int shade){
		System.out.println("Color: "+color+" has been used to draw with opacity: "+shade);
	}
	
	public static void main(String[] args) {
		MyInterface myInterface = StaticMethodReference::drawPainting;
		myInterface.draw("Red", 5);
	}
}

Instance Method Reference for an object of a particular type

The method in the interface and the instance method in the class should have the equivalent arguments.

The class should be static inner class 

public abstract interface MyInterface{
	public String draw();
}
public class InstanceMethodReferenceObjectType {
	public static void main(String args[]) {
        final List<Person> people = Arrays.asList(new Person(), new Person());
		people.forEach(Person::printName);
    }
    private static class Person {
        public String printName() {
			return "";
        }
    }
}

Instance Method Reference for an existing object

The method in the interface and the instance method in the class should have the equivalent arguments. 

public abstract interface MyInterface{
	public void draw(String color, int shade);
}
public class InstanceMethodReferenceExistingObject {
	public void drawPainting(String color, int shade){
		System.out.println("Color: "+color+" has been used to draw with opacity: "+shade);
	}
	public static void main(String[] args) {
		InstanceMethodReferenceExistingObject obj = new InstanceMethodReferenceExistingObject();
		MyInterface myInterface = obj::drawPainting;
		myInterface.draw("Green", 15);
	} 
}

Constructor Reference

public class ConstructorReference
{
    public static void main(String[] args)
    {
        Supplier<List<String>> st1 = ArrayList::new;
    }
}
