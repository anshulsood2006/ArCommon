Access Specifiers

Features

  • They are the keywords to restrict the scope of a class, constructor, method, local variable, member variable or class variable
  • There are 4 types of access specifiers
    Access SpecifierMeaning
    DefaultVisible only inside the package
    PublicVisible anywhere
    ProtectedVisible inside the package and all subclasses. In order to be able to access protected entity we need to subclass the class where the protected entity is declared.
    PrivateVisible only inside class

Examples

Entities with all the above four types of access specifiers are available inside the same class

public class MyTestClass {
	public void myPublicMethod(){
		System.out.println("myPublicMethod");
	}
	
	void myDefaultMethod(){
		System.out.println("myDefaultMethod");
	}
	
	private void myPrivateMethod(){
		System.out.println("myPrivateMethod");
	}
	
	protected void myProtectedMethod(){
		System.out.println("myProtectedMethod");
	}
	
	public static void main(String[] args) {
		MyTestClass myTestClass = new MyTestClass();
		myTestClass.myDefaultMethod();
		myTestClass.myPublicMethod();
		myTestClass.myProtectedMethod();
		myTestClass.myPrivateMethod();
	}
}

Entities with public, default and protected specifiers ONLY are available inside any class in the same package. Entities with private specifier are NOT.

public class TestAccessSpecifiersSamePackage {
	
	public static void main(String[] args) {
		MyTestClass myTestClass = new MyTestClass();
		myTestClass.myDefaultMethod();
		myTestClass.myPublicMethod();
		myTestClass.myProtectedMethod();
		myTestClass.myPrivateMethod(); // Compile Error "The method myPrivateMethod() from the type MyTestClass is not visible"
	}
	
}

Entities with public specifier ONLY are available inside any class in the different package. Entities with default, private and protected specifier are NOT.

public class TestAccessSpecifierOtherPackageSubclass extends MyTestClass{
	
	public static void main(String[] args) {
		TestAccessSpecifierOtherPackageSubclass myTestClass = new TestAccessSpecifierOtherPackageSubclass();
		myTestClass.myDefaultMethod(); // Compile Error "The method myDefaultMethod() from the type MyTestClass is not visible"
		myTestClass.myPublicMethod();
		myTestClass.myProtectedMethod(); // Compile Error "The method myProtectedMethod() from the type MyTestClass is not visible"
		myTestClass.myPrivateMethod(); // Compile Error "The method myPrivateMethod() from the type MyTestClass is not visible"
	}
	
}

Entities with public and protected specifier ONLY are available inside any subclass of the class (having those methods) in the different package. Entities with default and private specifier are NOT.

public class TestAccessSpecifierOtherPackageSubclass extends MyTestClass{
	
	public static void main(String[] args) {
		TestAccessSpecifierOtherPackageSubclass myTestClass = new TestAccessSpecifierOtherPackageSubclass();
		myTestClass.myDefaultMethod(); // Compile Error "The method myDefaultMethod() from the type MyTestClass is not visible"
		myTestClass.myPublicMethod();
		myTestClass.myProtectedMethod();
		myTestClass.myPrivateMethod(); // Compile Error "The method myPrivateMethod() from the type MyTestClass is not visible"
	}
	
}
