Blocking Queue

Features

  • This is thread safe queue in Java.
  • The size needs to be provided in the constructor and is mandatory.
  • It support operations that wait for the queue to become non-empty when retrieving and removing an element, and wait for space to become available in the queue when adding an element.
  • Null objects can not be added and throws Null Pointer Exception if such thing is tried.
  • This is primarily used for implementing producer consumer problem.
  • The methods are:
    1. put(): Add the element to the tail of queue. This method will wait in case queue is full.
    2. offer(): Add the element to the tail of queue.
    3. take(): Retrieve and remove an element at the head of the queue. This method will wait in case queue is empty.
  • It has below implementation classes which are used to store elements in FIFO order and also provides thread safe operations.
    1. Array Blocking Queue: It uses an internal array in which the elements are kept. Because it uses an array, it has a fixed size which is given in the constructor.
    2. Linked Blocking Queue: It uses nodes (like a linked list), to keep track of the order of the elements, which increases the complexity of the data structure. It can have a fixed-size limit as well, but if we don't define one the limit is Integer.MAX_VALUE by default.
    3. Priority Blocking Queue
    4. Synchronous Queue.html
