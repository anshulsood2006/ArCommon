Features

Bit

It stands for binary digit and is a basic unit of information used in computing and digital communications.

Byte

It is also a unit of information used in computing and digital communications and is equal to 8 bits.

Autoboxing

Convert primitive data types to corresponding Wrapper classes. This can be done in below ways:

  • Passing as a parameter to a method that expects an object of the corresponding wrapper class. For example a method with Integer argument can be called by passing int, java compiler will do the conversion of int to Integer.
  • Assign to a variable of the corresponding wrapper class. For example, assigning a Long object to long variable.

Unboxing

Convert Wrapper classes to corresponding primitive data types. This can be done in below ways:

  • Pass as a parameter to a method that expects a value of the corresponding primitive type.
  • Assign to a variable of the corresponding primitive type.

Association:

It is a type of HAS-A relationship between two classes. It is of two types:

  1. Aggregation: It is a one way relationship between two classes. Both the entries can survive individually which means ending one entity will not effect the other entity.
  2. Composition: Both the entities are dependent on each other. The composed object cannot exist without the other entity.

Comparator and Comparable: These are the interfaces used to sort the object of a user defined collection.

Frequently Asked Questions

What is the difference between equals and ==?

  • == is a operator while equals is a method in Java.
  • == behaves identical for all the objects where as equals being a method can be overridden to compare the objects based on business needs.
  • == can be used to compare both the primitives and objects where as the equals can be used to compare objects only
  • For primitives, == operator will return true only in case the values are same otherwise this will return false.
  • For objects, == operator will return true only if the memory reference is same for the objects otherwise this will return false.

What is default return value for hashcode() method inside the Object class

The value returned by the default implementation of hashCode() is called identity hash code and it is by default the integer representation of the memory address of the object. However this implementation is JVM dependent. Even if the hashCode method is overridden, the identity hash code can be obtained for the object as System.identityHashCode(object)

What is the contract between equals and hashcode method in Java

Objects which are equals must also return the same hashcode. Both equals() and hashCode() methods are defined in the Object class as below:

  • public native int hashCode()
  • public boolean equals(Object obj) {
	return (this == obj);
}

As per the contract, whenever hashCode() method is invoked on the same object more than once during an execution of a Java application, the hashCode method must consistently return the same integer. But normally JVM will relocate the objects, after GC or compaction.

What is the difference between composition and aggregation?

  • Aggregation implies a relationship where the child can exist independently of the parent. For example, Bank and Employee, delete the Bank and the Employee still exist. whereas Composition implies a relationship where the child cannot exist independent of the parent. Example: Human and heart, heart don't exist separate to a Human
  • Aggregation relation is "has-a" and composition is ""part-of" relation.
  • Composition is a strong association whereas aggregation is a weak association.

What is the difference between local variable and member variable

A variable declared inside a method is called local variable while variable declared inside a class is called a member variable

What is the need of String pool in Java?

Since the strings are immutable in Java, so in order to save memory on creation of same string objects again and again, Java instead uses the string objects from string pool.

What other datatypes other than String are cached in Java?

If the value p being boxed is true, false, a byte, or a char in the range \u0000 to \u007f, or an int or short number between -128 and 127 (inclusive), then let r1 and r2 be the results of any two boxing conversions of p. It is always the case that r1 == r2.
