Functional Interface

Features

  • This concept was introduced in Java 8 to support functional programming.
  • An interface having one and only one abstract method is called a functional interface
  • A functional interface can have any number of default concrete method.
  • A functional interface should be annotated with annotation @FunctionalInterface .
  • An instance of functional interface can be created using lambda expression.
  • A lambda expression can be used to avoid bulky anonymous implementation of a functional interface.

Examples

  • java.lang.Runnable having one abstract method void run()
  • java.awt.event.ActionListener having one abstract method void actionPerformed(ActionEvent e)
  • java.lang.Comparable<T> having one abstract method int compareTo(T o)
  • java.util.function.Predicate having one abstract method boolean test(T t)
  • java.util.function.BinaryOperator having one abstract method T apply(T x, T y)
  • java.util.function.Function having one abstract method R apply(T t)
  • java.util.function.Supplier having one abstract method T get()
  • java.util.function.Consumer having one abstract method void accept(T t)
