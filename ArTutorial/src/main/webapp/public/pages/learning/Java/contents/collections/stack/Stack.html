Stack

Features

  • It is an ordered list of objects with its use limited to inserting as well as deleting the elements from the top of list.
  • New elements are inserted at the top of the stack, and the stack retrieval operations also obtain elements from the top of the stack.
  • It works on the principle of last in first out. (LIFO)
  • The main operations that can be performed on stack are:
    1. push(): Adds element at the top of stack
    2. peek(): Returns the top element of stack but the element is not removed from stack. Throws EmptyStackException if stack is empty.
    3. pop(): Returns the top element of stack and also the element is removed from queue. Throws EmptyStackException if stack is empty.
    4. empty(): Returns true is stack is empty else returns false.
    5. search(Object element): Returns the position of element in the stack if present, else returns -1.
