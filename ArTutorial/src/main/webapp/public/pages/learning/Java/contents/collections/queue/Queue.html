Queue

Features

  • It is an ordered list of objects with its use limited to inserting elements at the end of list and deleting elements from the start of list
  • New elements are inserted at the tail of the queue, and the queue retrieval operations obtain elements from the head of the queue.
  • It works on the principle of first in first out. (FIFO)
  • The main operations that can be performed on queue are:
    1. add(): Adds element at the tail of queue i.e. last of linked list or as per priority in case of priority queue.
    2. peek(): Returns the head element of queue but the element is not removed from queue. Null returned if queue is empty
    3. element(): Returns the head element of queue but the element is not removed from queue. Throws NoSuchElementException if queue is empty
    4. poll(): Returns the head element of queue and also the element is removed from queue. Null returned if queue is empty
    5. remove(): Returns the head element of queue and also the element is removed from queue. Throws NoSuchElementException if queue is empty
  • The main implementation classes are:
    1. Linked List: Used to store elements in FIFO manner in the order of insertion.
    2. Priority Queue: Used to store elements according to some priority.
    3. Blocking Queue: Interface whose implementation classes are used to store elements in FIFO order and also provides thread safe operations.
