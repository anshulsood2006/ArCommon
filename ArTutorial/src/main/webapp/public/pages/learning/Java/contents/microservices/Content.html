Microservices

Features of a Microservice

  • It is a service oriented architecture of software development.
  • It is a software architecture wherein each component of an application is build as an independent autonomous service.
  • Each microservice performs a unique business goal.
  • The microservices communicate with each other in stateless mode where each request/response is independent of the other.
  • The microservices are exposed by REST.
  • Various microservices communicate with each other by using lightweight protocols such as HTTP or REST or messaging protocols
  • A microservice can be created using Spring Boot in Java.

Features of a Microservice

  • A microservice never serves more than one responsibility.
  • A microservice never restricts itself from accepting appropriate technology stack or database.
  • A microservice failure never causes complete failure of large application. Only affected module gets impacted.
  • A microservice is always automatically deployable.

What are the differences between monolitic and microservice architecture?




MonolithicMicroservice


A single code base for all the business objective.Each business goal is delivered by an individual microservice.


Application takes much time to start.Each microservice takes very less time to start.


Since code is tightly coupled, changes are difficult to do.Since code is loosely coupled, changes are rather easy to do.


If a feature is not working whole application needs to be rebuild after fix. The complete system goes down, so fault isolation is difficult.If a feature is not working, only that microservice needs to be rebuild after fix. The complete system does not go down, so fault isolation is easy.


Resources need to be added to complete application for scaling purpose.Resources need to be added to the only that microservice server where load is high.


The development and management is tough as the team size is comparatively large.The development and management is easy as the team size is small.


The complete application is build on one technology only.Different technologies can be used for different microservices


Change in data model affects the entire database.Change in data model affects the database model for that particular webservice only.




What is Service Virtualization?


It is the method to emulate the behaviour of microservice for testing purpose so they one component can be tested even when the other key components are not ready.
