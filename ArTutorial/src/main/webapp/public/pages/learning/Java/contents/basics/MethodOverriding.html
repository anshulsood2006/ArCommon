Method Overriding

Features

  • Methods overriding is also called dynamic polymorphism or run time binding
  • Two methods are to be called overloaded if
    • Both methods have the same signature but different implementations
    • The methods are present in base and child class respectively
  • Method can be overridden by adding keyword final to overridden method in child class. 
    	public class ClassA{
		public String method(){
			return "ClassA :: method()";
		}
	}
	
	public class ClassB extends ClassA{
		//No Compilation Error
		public final String method(){//Overridden method by adding final keyword
			return "ClassB :: method()";
		}
	}
  • Final method in parent class can not be overridden in child class. The below code will give error :"Cannot override the final method from ClassA" 
    	public class ClassA{
		public final String method(){
			return "ClassA :: method()";
		}
	}
	
	public class ClassB extends ClassA{
		//Compilation Error: "Cannot override the final method from ClassA"
		public String method(){//Trying to override final method
			return "ClassB :: method()";
		}
	}
  • Private method in parent class can not be overridden in child class. There will be no compilation errors though.
  • Method can not be overridden by reducing the visibility of overridden method in child class. The below code will give error :"Cannot reduce the visibility of the inherited method from ClassA" 
    	public class ClassA{
		public String method(){
			return "ClassA :: method()";
		}
	}
	
	public class ClassB extends ClassA{
		//Compilation Error: Cannot reduce the visibility of the inherited method from ClassA
		private String method(){  //Access specifier has been reduced from public o private in the method signature
			return "ClassB :: method()";
		}
	}
  • Method can not be overridden by removing keyword static from the overridden method in child class. The below code will give error :"This instance method cannot override the static method" 
    	public class ClassA{
		public static String method(){
			return "ClassA :: method()";
		}
	}
	
	public class ClassB extends ClassA{
		//Compilation Error: This instance method cannot override the static method from ClassA
		public String method(){  //static keyword has been removed from the method signature
			return "ClassB :: method()";
		}
	}
  • Method can not be overridden by adding keyword static to the overridden method in child class. The below code will give error: "This static method cannot hide the instance method from ClassA" 
    	public class ClassA{
		public String method(){
			return "ClassA :: method()";
		}
	}
	
	public class ClassB extends ClassA{
		//Compilation Error: This static method cannot hide the instance method from ClassA
		public static String method(){  //static keyword has been added to the method signature
			return "ClassB :: method()";
		}
	}
  • Method can not be overridden by changing the return type in the overridden method in child class. The below code will give error: "The return type is incompatible with ClassA.method()" 
    	public class ClassA{
		public String method(){
			return "ClassA :: method()";
		}
	}
	
	public class ClassB extends ClassA{
		//Compilation Error: The return type is incompatible with ClassA.method()
		public int method(){  //return type has been changed in  the method signature
			return 5;
		}
	}

Examples

Method overriding

public class ClassA{
	public String method(){
		return "ClassA :: method()";
	}
}

public class ClassB extends ClassA{
	public String method(){
		return "ClassB :: method()";
	}
}

public class MethodOverriding {
	
	public static void main(String[] args) {
		ClassA a = new ClassA();
		System.out.println(a.method());
		
		a = new ClassB();
		System.out.println(a.method());
	}
	
}

Frequently Asked Questions

Can we change the access specifier of the method from public to private in child class in case of method overriding?

No this will give compile time error "Cannot reduce the visibility of the inherited method from base class"
