SOLID Principles

Features

  • Single Responsibility: Every class should have a single responsibility.
  • Open Closed: Every class should be open for extension but closed for modification.
  • Liskov Substitution: Derived types must be completely substitutable for their base types. e.g. a rectangle can be replaced by a square
  • Interface Segregation: Clients should not be forced to implement unnecessary methods which they will not use
  • Dependency Injection: Higher level modules should not depend on low level modules. Depend on abstractions, not on concretions e.g. use of Stream interface in place of concrete implementations such as Filestream or Stringstream. Both should depend on abstractions.
