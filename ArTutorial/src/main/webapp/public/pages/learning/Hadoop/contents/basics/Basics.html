Hadoop

Features

  • Hadoop is an open source framework to store, process and analyze huge volume of data.
    • Store: For storage, hadoop uses Hadoop Distributed File System using commodity hardware to create clusters. It works on write once and read many times principle.
    • Process: For processing, hadoop uses MapReduce.
    • Analyze: For analyzing, hadoop uses hive or pig.
  • The three V's of Big data are:
    • Velocity
    • Volume
    • Variety
  • Modules of Hadoop
    • HDFS: Hadoop Distributed File System states that the files will be broken into blocks and stored in nodes over the distributed architecture.
    • Yarn: It stands for yet another resource negotiator and is used for job scheduling and managing the hadoop cluster. It sits between HDFS and processing engine in a clustered environment.
    • Map Reduce: It is a framework to process huge amount of data in parallel. It is divided into two stages:
      • Map task: It takes input data and converts it into a data set which can be computed in Key value pair. It involves the splitting and mapping of input data.
      • Reduce task: It takes the output of map task and combines those data tuples based on the key and accordingly modifies the value of the key.
    • Hadoop Common: It contains java apis to start hadoop and apis used by other hadoop modules.

  • Hadoop Configuration files
    • core-site.xml contains configuration of NameNode in the cluster
    • hdfs-site.xml contains configuration of replication factor, block size, permissions, data directory, name node directory for HDFS.
    • mapred-site.xml contains configuration of map reduce alogithms and resource managers for hadoop.
    • yarn-site.xml contains configuration of map reduce alogithms and resource managers for hadoop.

Frequently Asked Questions


What is the difference between NameNode and DataNode in hadoop?


The main difference between NameNode and DataNode in Hadoop is that the NameNode is the master node in HDFS that manages the file system metadata while the DataNode is a slave node in HDFS that stores the actual data as instructed by the NameNode.
