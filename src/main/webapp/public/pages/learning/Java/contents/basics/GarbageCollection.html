Garbage Collection

Features

  • The destruction of unused/unreachable objects by a daemon thread is called garbage collection.
  • It frees heap memory in JVM.
  • An object is said to be unreachable when there is no reference to it. e.g.
    Integer i = new Integer(4);
    i = null;
    Integer(4) object is unreachable now and can be garbage collected.
  • One of the basic way of garbage collection involves three steps:
    1. Marking: In this step garbage collector identifies which objects are in use and which ones are not in use.
    2. Normal Deletion: Garbage Collector removes the unused objects and reclaim the free space to be allocated to other objects.
    3. Deletion with Compacting: For better performance, after deleting unused objects, all the survived objects can be moved to be together. This will increase the performance of allocation of memory to newer objects.
  • Types of Garbage Collection:
    1. Serial GC (-XX:+UseSerialGC)
    2. Parallel GC (-XX:+UseParallelGC)
    3. Parallel Old GC (-XX:+UseParallelOldGC)
    4. Concurrent Mark Sweep (CMS) Collector (-XX:+UseConcMarkSweepGC)
    5. G1 Garbage Collector (-XX:+UseG1GC)
