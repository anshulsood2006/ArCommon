Queue

Features

  • It is an ordered list of objects with its use limited to inserting elements at the end of list and deleting elements from the start of list
  • The main operations that can be performed on queue are:
    1. add(): Adds element at the tail of queue i.e. last of linked list or as per priority in case of priority queue.
    2. peek(): Gives the head element of queue but the element is not removed from queue. Null returned if queue is empty
    3. element(): Gives the head element of queue but the element is not removed from queue. Throws NoSuchElementException if queue is empty
    4. poll(): Gives the head element of queue and also the element is removed from queue. Null returned if queue is empty
    5. remove(): Gives the head element of queue and also the element is removed from queue. Throws NoSuchElementException if queue is empty
  • The main implementation classes are:
    1. Linked List: Used to store elements in FIFO manner in the order of insertion.
    2. Priority Queue: Used to store elements according to some priority.
    3. Blocking Queue: Used to store elements in FIFO order but provides thread safe operations.
