How to avoid singleton destruction on using readResolve method for serialized class?

One a singleton class is declared as serializable, a new instance can be created using readObject() method which always return a new instance. To avoid this, override the readResolve() method to return the singleton instance instead. 

	private Object readResolve(){
        	return INSTANCE;
    	}
