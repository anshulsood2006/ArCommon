Enums

  1. An enum type is a subclass of abstract class java.Lang.Enum
  2. Enums are used to group named constants.
  3. They are used when all the possible values are known at compile time e.g. days of week, months of year etc.
  4. They can have variables, methods and constructors.
  5. We can declare abstract methods in java enum, then all the enum fields must implement the abstract method
  6. Enums can be used in switch statement,
  7. Every enum is internally implemented by using Class. e.g 
    enum Color
{
	RED, GREEN, BLUE;
}
    is internally represented as: 
    class Color
{
     public static final Color RED = new Color();
     public static final Color BLUE = new Color();
     public static final Color GREEN = new Color();
}
  8. Every enum constant represents an object of type enum.
  9. Enum type can be passed as an argument to switch statement.
  10. All enums implicitly extend java.lang.Enum class. As a class can only extend one parent in Java, so an enum cannot extend anything else.
  11. toString() method is overridden in java.lang.Enum class,which returns enum constant name.
  12. Enum can implement many interfaces.
  13. Enum can contain constructor and it is executed separately for each enum constant at the time of enum class loading
  14. Every enum constant is always implicitly public static final. Since it is static, we can access it by using enum Name. Since it is final, we cannot create child enums.
  15. Constructor is always private or package-private. If you try to make constructor as public you get a compilation error "Illegal modifier for the enum constructor; only private is permitted."
  16. Since enum constants can not be created explicitly so constructor can not be called.
  17. By default enums have their own string values. e.g. for Example 1 below the value for enum YES is YES and that for enum NO is NO, while for Example 2 the values of variable default value and size in bits for enum BOOLEAN are false and 1 respectively
  18. Parameterrized enums have at least one getter method to get the value of the enum.
  19. Enum in java implicitly implements both Serializable and Comparable interfaces

Examples

Example 1:

	public enum ArOptions {	
		YES, NO
	}

Example 2:

	//Declare the enumClass
	public enum ArDataTypeEnum {
	
		//Declare all the named enumConstant
		BOOLEAN("false", 1),
		BYTE("0", 8),
		CHAR("\u0000", 16),
		SHORT("0", 16),
		INT("0", 32),
		FLOAT("0.0", 32),
		LONG("0", 64),
		DOUBLE("0.0", 64);
		
		//Declare each of the variable involved for parameterized enum
		private String defaultValue;
		private int sizeInBits;
		
		//Create private constructor to initialize each of the variable involved for parameterized enum
		private ArDataTypeEnum(String defaultValue, int sizeInBits){
			this.defaultValue = defaultValue;
			this.sizeInBits = sizeInBits;
		}
		
		//Define public getter methods for each of the variable involved for parameterized enum
		public String getDefaultValue() {
			return this.defaultValue;
		}
		public int getSizeInBits() {
			return this.sizeInBits;
		}
	}

Frequently Asked Questions

How to get all the values inside enum?

{enumClass}.values() returns an array of all the enum constants defined in an Enum

How to get the name of enum?

{enumConstant}.name() returns the name of enum constant defined in an Enum e.g. ArDataTypeEnum.SHORT.name() in Example 2 above will return SHORT

How to find if an enum constant is present inside an enum?

If {enumClass}.valueOf(name) does not through IllegalArgumentException and returns enum constant.

What is the return value of {enumConstant}.ordinal() method?

This method returns the order in which the enumConstant is declared in the enumClass e.g. ArDataTypeEnum.SHORT.ordinal() in Example 2 returns 3

What is the difference between comparing Enums using equals and == and compareTo methods?

== and equals return the same result since the equals method is internally using == only. 

	public final boolean equals(Object other) {
        	return this==other;
    	}

This method is declared final to avoid any overriding. We should ideally be using == methods to avoid NullPointerException which may occur in case of using equals obj1 == obj2 can never throw this exception but obj1.equals(obj2) can. Also == method provides type safety during compile time.

What is advantage and disadvantage of using Enum as Singleton?

  • Enums are easy to write
  • They handle serialization by themselves. JVM takes care of singleton destruction which can be caused by calling readObject() method.
  • Enum constants are also thread safe

How to convert string to enum?

A string can be converted to enum in below two ways:

  • Enum.valueOf(enumValue)
  • {EnumClass}.valueOf(enumValue)
