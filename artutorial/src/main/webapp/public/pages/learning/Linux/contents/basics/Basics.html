Linux Basics

Features

  • Commands
    CommandMeaning
    uname -rGet the version of linux installed.
    yum repolist -vList the yum repositories available
    yum clean packagesRemove the cached packages
    cd ~Go to home directory.
    :42Go to line number 42 in vi.
    find /dir -name fileNameSearch for the file with name fileName inside directory /dir.
    find /dir -iname fileNameSearch for the file with name fileName inside directory /dir without regard for the text case.
    suChange to root user
    usermod -aG wheel {username}Add user to sudoer file
    sudo yum updateInstall latest updates on red hat
    sudo yum install java-1.8.0-openjdk-develInstall open jdk
    sudo yum remove java-1.8.0-openjdk-develUninstall open jdk
    cat /dev/randomProduces random device
    cat /dev/nullProduces a device that produces no output. We can pipe the output of a command to /dev/null to discard the command.
    printenvPrints all the environment variables
    tar -xvf {tarfile}.tarUntar a tar file.
  • Directory Structure
    • / root directory.
    • /bin, /usr/bin, /sbin contains essential binaries.
    • /boot contains the files needed for system boot.
    • /dev contains devices files.
    • /etc contains the system wide configuration files.
    • /home contains folder for each user.
    • /lib contains the essential libraries needed by essential binaries inside /bin, /usr/bin, /sbin.
    • /lost+found contains the file for recovery after system crash.
    • /media contains the removable media devices.
    • /mnt contains the temporary mount points.
    • /opt contains the optional packages.
    • /proc contains the Kernel & Process Files.
    • /root contains the home directory of root user. Note that this is not located inside /home directory like home directories of other users
    • /run contains the transient files require by applications like sockets and process IDs.
    • /tmp contains the temporary files.
    • /usr contains the applications and files used by users. E.g. /usr/lib/jvm contains Java
    • /var contains the writable data produced by applications inside the /usr.
  • Install Java
    • Download tar file for the Oracle's website .
    • mkdir /usr/lib/jvm
    • cd /usr/lib/jvm
    • tar -xvzf ~/Downloads/{jdk}.tar.gz
    • vi /etc/bashrc and add path as PATH="$PATH:/usr/lib/jvm/{jdk_directory}/bin:/usr/lib/jvm/{jdk_directory}/jre/bin:"
    • vi /etc/bashrc and add JAVA_HOME as JAVA_HOME="/usr/lib/jvm/{jdk_directory}"
    • vi /etc/bashrc and add JRE_HOME as JRE_HOME="/usr/lib/jvm/{jdk_directory}/jre"
    • source /etc/bashrc
  • Install Maven
    • Download tar file from the Apache's website.
    • cd /opt
    • tar -xvzf ~/Downloads{maven}.tar.gz
    • vi /etc/bashrc and add path as PATH="$PATH:/opt/{mvn_directory}/bin:"
    • vi /etc/bashrc and add M2_HOME as M2_HOME=/opt/{mvn_directory}"
    • source /etc/bashrc

Frequently Asked Questions

How to change to root user in linux?

use command "su"

How to start a shell script with some name?

Edit the shell script file and use shebang (the very first line) to name the process e.g. #!/bin/bash /home/pi/infinite_loop.bash

How to find and remove a package in redhat?

  • Find the installed package using rpm -qa | grep {program}
  • Remove the package using yum remove program package}

How to fix error locate: can not stat () `/var/lib/mlocate/mlocate.db'

Update the binary database using below command sudo updatedb

How to fix error file {file} from install of package {packageName} conflicts with file from package {packageName}

Take an example of error file /usr/share/man/man1/docker-attach.1.gz from install of docker-ce-cli-1:19.03.5-3.el7.x86_64 podman-manpages-1.4.2-6.module+el8.1.0+4830+f49150d7.noarch found during installation of docker of RHEL8. To fix the above error, remove the package on the right hand side of the error message i.e. yum remove podman-manpages.
